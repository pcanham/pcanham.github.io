Simple use of netcat for checking a port is open on a firewall/load balancer

January 27, 2009

Server (Listener)

root@server:/<span class="code-keyword">var</span>/tmp> cat pcanham.txt
Welcome to the Server on port 7735.
root@server:/<span class="code-keyword">var</span>/tmp> netcat -l -s server.example.com -p 7735 <pcanham.txt
![](http://www.thinkside.co.uk/blog/images/warning.gif) once something has connected the app will stop you will need to put it into a loop
Option Desc
-l listener
-s source address (IP address to bind to)
-p port

Client

root@scar:/> telnet server.example.com 7735
Trying 10.10.10.20...
Connected to server.example.com.
Escape character is '^]'.
Welcome to the Server on port 7735.
^]
telnet> q
Connection to server.example.com closed.
